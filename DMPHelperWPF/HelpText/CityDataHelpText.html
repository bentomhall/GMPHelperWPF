CityData.json Format

This config file contains information about the major cities of the setting. The generated minor settlements will take their naming and demographic cues from the nearest big city from this list.

Element Format and necessary keys

This file (like all the rest), is a list of mappings. The keys must be spelled exactly as written (including capitalization), but the order doesn't matter.
Key Data Type Comments
Name string A unique identifier used to connect to other files.
Nation string This must match a value declared in the Nation field of the Culture file.
Population int
Size string A description such as city, capital, sprawling, etc.
Region string This must match a value declared in the Name field of the Regions file.
Terrain string
Races {string : decimal} Each key is a race name that must match a value declared in the Race field of the Culture file. The values are the decimal abundance of that race in the city and its surroundings.
Tech int 0 ≤ x ≤ 4. This sets the general cap on what goods and services are available near this city. See Items file for more discussion.
Prefixes list(string) Settlement names will be generated from these three lists as prefix + combiner + infix + suffix. If the names should only have 2 parts (or 1), leave the prefixes (and then infixes) fields as empty lists [].
Infixes list(string) See above.
Suffixes list(string) See above.
Combiner string See above.

JSON format

Data types are set out in italics like string. The relevant data types are:

Data Type Meaning Example
list(type) A comma separated list of that data type. Lists start and end with square brackets. ["item 1", "item 2"]; [1, 2, 3, 4]
string A double-quoted piece of information that can contain any UTF-8 character. "a value"
int A whole number, written without quotes. If the range is limited, it's noted as start ≤ int ≤ stop (including both endpoints) 42
decimal A number with a decimal point. In this app, that will always be in the range 0.0 ≤ x ≤ 1.0 and will mean a probability of occurance. 0.5
{string : type} A mapping between a string (the key) and a value of type. Written as a comma-separated list of "key":value, enclosed in curly braces. The colon is important. {"This": 1, "That": 2}
