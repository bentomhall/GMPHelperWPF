races.json Format

This config file contains information about the races and cultures of the setting. Each unique combination of race and nation should be represented; if not, the program will crash when it tries to generate a missing combination.

Element Format and necessary keys

This file (like all the rest), is a list of mappings. The keys must be spelled exactly as written (including capitalization), but the order doesn't matter.
Key Data Type Comments
Race string A unique identifier used to connect to other files.
Nation string This must match those defined in the Nations config file.
Culture list(string) These must be present in the Name config file--one will be selected from this list at random to assign a name.
Religiosity list(string) A list of religious descriptors. I use devout (devoutly worshipping one or more God), casual (propitiates or otherwise casually worships one or more God), ancestor worship (reveres ancestor spirits), animist (worships elemental or natural spirits), and agnostic (not particularly religious).
Gender {string:decimal} The gender balance of the race. The only two required keys are Male and Female, and the decimal values can be set to 0 if desired. This sets which pool of names are used for that NPC.
Subrace {string: decimal} The distribution of notable subraces. Use N/A as a key for the base race if that's an option.

JSON format

Data types are set out in italics like string. The relevant data types are:

Data Type Meaning Example
list(type) A comma separated list of that data type. Lists start and end with square brackets. ["item 1", "item 2"]; [1, 2, 3, 4]
string A double-quoted piece of information that can contain any UTF-8 character. "a value"
int A whole number, written without quotes. If the range is limited, it's noted as start ≤ int ≤ stop (including both endpoints) 42
decimal A number with a decimal point. In this app, that will always be in the range 0.0 ≤ x ≤ 1.0 and will mean a probability of occurance. 0.5
{string : type} A mapping between a string (the key) and a value of type. Written as a comma-separated list of "key":value, enclosed in curly braces. The colon is important. {"This": 1, "That": 2}
